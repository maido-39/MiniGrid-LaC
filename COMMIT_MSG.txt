TL;DR: API 단에서 carrying object 터미널 출력 기능 추가, gen_obs() 호출 시 리스트 호환성 버그 수정, 모든 디버깅 코드 제거

## 주요 변경 사항

### 기능 추가
- **API 단 carrying object 출력 기능**
  - `CustomRoomEnv`에 `_format_carrying_object()`, `_format_single_carrying_object()`, `_print_carrying_status()` 메서드 추가
  - pickup/drop 성공 시 터미널에 `[API] Carrying Objects (count): [obj1, obj2, ...]` 형식으로 출력
  - 리스트/단일 객체 모두 지원하며, emoji와 색상 정보 포함

### 버그 수정
- **gen_obs() 호출 시 리스트 호환성 문제 해결**
  - `self.carrying`이 리스트일 때 `gen_obs()` 호출 시 발생하는 `'list' object has no attribute 'encode'` 에러 수정
  - pickup/drop 성공 후 `gen_obs()` 호출 전에 `self.carrying`을 임시로 단일 객체로 변환하고, 호출 후 리스트로 복원
  - 의도하지 않은 움직임 문제 해결

### 호환성 개선
- **scenario2_keyboard_control.py 리스트 지원**
  - `env.carrying`이 리스트인 경우를 처리하도록 수정
  - 다중 객체 carrying 상태를 올바르게 표시

### 코드 정리
